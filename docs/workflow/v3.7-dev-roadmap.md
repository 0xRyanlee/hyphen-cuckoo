## 二、v3.7 開發優先順序

### P0 — 必做（UX 正確性問題）

#### [P0-1] 打印中心 WYSIWYG 重構
**問題：** 目前模板以 JSON 手動輸入，使用者無法理解元素結構。  
**目標：** 拖拽排序元件列表 → 即時熱敏紙預覽 → 點擊打印測試。  

實作步驟：
1. 將 `ElementToggleCard` + `ElementEditorFields` 從 `marketing-page.tsx` 抽取為獨立 component：`src/components/print-element-editor.tsx`
2. `print-templates-page.tsx` 左側 JSON 輸入改為 element card list（與 marketing-page 同結構）
3. 右側 live preview 保持不變（已有 `renderLivePreview`）
4. 加入「測試打印」按鈕：呼叫 `print_test_ticket` command（已存在）
5. 刪除 JSON textarea（僅保留「進階 JSON 編輯」collapsible 折疊）

驗收：使用者可在不看 JSON 的情況下完成模板建立與元件配置。

---

#### [P0-2] 营銷中心 Receipt Tab 從後端載入真實模板
**問題：** 目前 `receiptElements` 使用 `DEFAULT_RECEIPT_ELEMENTS` 靜態資料，不反映實際模板。  
**目標：** 頁面載入時抓取預設 receipt 模板，在唯讀預覽中顯示真實元素。  

實作步驟：
1. 在 `MarketingPage` 的 `useEffect` 中新增：
   ```ts
   invoke<PrintTemplate[]>("get_print_templates", { templateType: "receipt" })
     .then(templates => {
       const def = templates.find(t => t.is_default) ?? templates[0];
       if (def) {
         const elems = JSON.parse(def.elements_json ?? "[]");
         setReceiptElements(elems);
       }
     })
     .catch(() => {});
   ```
2. `receiptElements` 改回 `useState<PrintElement[]>([])` 並補 setter
3. 唯讀 preview 在載入完成前顯示 skeleton

---

#### [P0-3] KDS 時區 Bug（Z suffix）
**問題：** `kds-page.tsx` 中訂單時間以 `2024-01-01T12:00:00Z` 格式存入，但 SQLite 儲存的是本地時間，加 Z 後解析成 UTC 導致顯示偏移 8 小時。  
**定位：** `src/pages/kds-page.tsx` 中所有 `new Date(... + "Z")` 呼叫。  
**修復：** 移除 Z suffix，改用 `new Date(str.replace("Z", ""))` 或在 Rust side 確保時間格式帶時區偏移。

驗收：KDS 訂單時間顯示與系統時鐘一致。

### P1 — 重要（功能正確性）

#### [P1-1] 退款記錄雙向同步
**問題：** `record_order_refund` 呼叫後，`orders-page` UI 不更新 `amount_paid` 顯示。  
**修復：** `handleCancelOrder` callback 後補 `loadData()` 重新拉訂單。

#### [P1-2] Print ticket 類型 badge 中文化
**問題：** Badge 顯示原始 type string（如 `kitchen_ticket`）而非中文。  
**修復：** 在 `print-tickets-page.tsx` 加 `typeLabel` mapping。

#### [P1-3] Settings 頁面 `window.alert` 殘留
**定位：** 全站掃描確認已無 `window.alert` / `window.confirm` 殘留（已全替換）。✅

### P2 — 體驗優化

#### [P2-1] 营销弹窗預覽整合到打印預覽
顧客下單後看到的 popup HTML 應與打印中心預覽視覺一致。  
在 `SurfacePanel` 的 previewHtml 使用 `.thermal-paper` 樣式渲染。

#### [P2-2] 打印測試 → 預覽對比
在 print-tickets-page 或 print-templates-page 加入「打印 & 預覽並排」視圖：左側 live preview，右側打印機輸出文字差異高亮。

#### [P2-3] 數據看板 — 色彩語意恢復
v3.6.x 統一改為 `text-foreground`，但統計數字失去語意區分（兌獎/發券/核銷）。  
建議使用帶 opacity 的 primary 色調作梯度：  
- 核銷（最重要）：`text-primary font-bold`  
- 發券：`text-primary/70`  
- 兌獎次數：`text-muted-foreground`
